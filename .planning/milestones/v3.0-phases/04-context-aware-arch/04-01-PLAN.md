  <behavior>
    - build_context_index(content) 扫描 cli.js 内容，识别文件中的主要功能区块/组件范围
    - detect_context(content, position: int) 根据字符串位置返回上下文标签列表
    - 区块识别基于已知模式（如函数定义、module 边界、字符串模式如 "tools"、"auth"、"mcp" 等组件标识符）
    - 对于无法识别的区块，返回 ["default"] 空上下文
    - 上下文标签使用简短英文标识符（如 "tools", "auth", "mcp", "status", "config"）
  </behavior>
  <action>
    先编写测试（RED），然后实现（GREEN）：

    1. 创建 tests/unit/test_context_detector.py 测试文件：
       - test_build_context_index_basic: 内容含明显组件边界时返回区块列表
       - test_build_context_index_empty: 空内容返回空索引
       - test_detect_context_known_region: 字符串位置在 "tools" 相关区域时返回 ["tools"]
       - test_detect_context_nested: 字符串位置在嵌套区域时返回多个标签（如 ["tools", "permission"]）
       - test_detect_context_default: 位置无法识别时返回 ["default"]
       - test_context_index_caching: 多次调用 detect_context 不重新扫描内容

    2. 创建 scripts/i18n/core/context_detector.py：
       - 定义 CONTEXT_PATTERNS 常量：正则模式列表，每个模式关联一个上下文标签
         ```python
         CONTEXT_PATTERNS = [
             (re.compile(r'tool_name|toolResult|tool_use'), "tools"),
             (re.compile(r'auth|login|token|credential'), "auth"),
             (re.compile(r'mcp_server|mcp_tool|McpServer'), "mcp"),
             (re.compile(r'status_bar|StatusItem'), "status"),
             (re.compile(r'config|settings|preference'), "config"),
             (re.compile(r'permission|approve|deny'), "permission"),
             (re.compile(r'agent|subagent|worker'), "agent"),
             (re.compile(r'commit|git|branch'), "git"),
             (re.compile(r'plan|yolo|auto_mode'), "mode"),
             (re.compile(r'cost|usage|billing|token'), "billing"),
         ]
         ```

       - build_context_index(content: str) -> list[dict]:
         - 扫描 content，找到所有 CONTEXT_PATTERNS 的匹配位置
         - 返回 [ {"tag": "tools", "start": 12345, "end": 23456}, ... ]
         - 使用 1024 字符的滑动窗口来扩展匹配的范围（一个匹配点前后各扩展 512 字符）
         - 相邻的同标签区块合并（间隔 <2048 字符的合并为同一区块）

       - detect_context(index: list, position: int) -> list[str]:
         - 遍历 index，检查 position 是否在某个区块的 [start, end] 范围内
         - 返回所有匹配的标签列表（一个位置可能在多个区块内）
         - 无匹配时返回 ["default"]
         - 注意：此函数接收预构建的 index，不从 content 重新扫描

       - 设计决策：build_context_index 一次性构建，detect_context 可被反复调用查询不同位置。
         这是因为 cli.js 有 13MB，不能每次替换都重新扫描。apply 流程中 build 一次，
         然后对每个匹配位置调用 detect。

    重要：这个模块只提供位置到标签的映射。具体如何在替换流程中使用由 Plan 02 实现。
  </action>
