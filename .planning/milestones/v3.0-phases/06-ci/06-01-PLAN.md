  <behavior>
    - count_translations: 读取 zh-CN.json，返回 {"total": int, "long": int, "medium": int, "short": int, "meta_version": str}
      - total = len(translations) 中所有条目数
      - long/medium/short 按字符串长度分类 (long>20, medium 10-20, short<10)，统计英文 key 的长度
    - load_baseline: 读取 coverage-baseline.json，返回 dict 或 None（文件不存在时）
    - compare_coverage: 对比当前统计与基线，返回 {"ok": bool, "current": dict, "baseline": dict, "diff": dict}
      - ok=true 当且仅当 current.total >= baseline.translated_count
    - main: CLI 入口，argparse 支持 --baseline-path（默认 .planning/coverage-baseline.json）和 --map-path（默认 scripts/zh-CN.json）
      - 输出 JSON 到 stdout
      - ok=true 退出码 0，ok=false 退出码 1
    - 边界情况: 基线文件不存在时输出警告但仍以 ok=true 退出（首次运行没有基线）
  </behavior>

  <action>

       - 测试 compare_coverage: current >= baseline 返回 ok=true
       - 测试 compare_coverage: current < baseline 返回 ok=false，diff 包含差值
       - 测试 load_baseline: 文件不存在返回 None
       - 测试 main: 完整 CLI 流程（使用 monkeypatch 临时文件）
       - 测试 main: 基线不存在时仍 ok=true

    2. 创建 scripts/ci_check_coverage.py:
       - 纯 Python 标准库（json, argparse, sys, pathlib）
       - count_translations(path): 读取 zh-CN.json，统计 translations 下所有 key 的数量和分类
       - load_baseline(path): 读取 JSON 基线文件
       - compare_coverage(current, baseline): 对比并生成报告
       - main(): argparse CLI，--baseline-path 和 --map-path 参数，JSON 输出到 stdout
       - 退出码: ok=true -> 0, ok=false -> 1

    3. 创建 .planning/coverage-baseline.json:

  <action>
    在现有 ci.yml 的 jobs 中新增 coverage-regression job（与 test job 平级，不依赖 test）:

    ```yaml
    coverage-regression:
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v4

        - name: Set up Python
          uses: actions/setup-python@v5
          with:
            python-version: "3.12"

        - name: Check translation coverage regression
          run: python3 scripts/ci_check_coverage.py --baseline-path .planning/coverage-baseline.json --map-path scripts/zh-CN.json
    ```

    关键设计点:
    - 独立 job，不依赖 test job，可以并行运行
    - 只用单一 Python 版本 (3.12)，无需矩阵（纯文件读取，无版本兼容风险）
    - checkout 整个仓库（需要 zh-CN.json + baseline 文件）
    - 脚本退出码 1 时 CI 自动失败
    - JSON 输出到 CI 日志中供开发者查看对比报告
  </action>
