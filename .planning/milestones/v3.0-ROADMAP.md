Claude Code CLI 汉化工具的完全重写。解决 v2.0 的备份污染、低覆盖率、无上下文感知等架构级问题，从零构建安全可靠的汉化引擎。

## Milestone Summary

**Key Decisions:**
- ✓ 完全重写（非修补）— 备份污染是架构级 bug
- ✓ 纯 Python 标准库，零外部依赖
- ✓ 增强型 regex（非 AST）— 13MB minified 文件不适合 AST
- ✓ 三级替换策略：长(全局) / 中(引号边界) / 短(词边界+skip)
- ✓ Hook sed 替换集成到 Python 引擎（hooks.py）
- ✓ 自动翻译采用纯规则引擎（词典→精确匹配→UI模板→动词模式→前缀后缀）
- ✓ coverage 命令：表格+JSON 双输出，按字符串长度分类

**Issues Resolved:**
- 备份污染 — v2.0 备份被中文污染导致无法恢复
- 路径检测 — v2.0 仅支持 npm 全局安装
- 短字符串误替换 — "Error" 等全局替换破坏运行时逻辑
- 版本更新后失效 — v2.0 无版本检测，CLI 更新后汉化丢失
- 无量化能力 — 无法知道翻译覆盖率

**Technical Debt:**
- auto_translate 模块覆盖率仅 63%（规则复杂度高）
- prefix_suffix_match 产生低质量翻译（已在本会话发现并绕过）
- CI 仅 GitHub Actions，未验证 macOS/Linux 差异
