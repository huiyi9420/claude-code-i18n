# Milestone v3.0: 完全重写 + 翻译增强

**Status:** SHIPPED 2026-04-08
**Phases:** 1-6
**Total Plans:** 17

## Overview

Claude Code CLI 汉化工具的完全重写与翻译增强。解决 v2.0 的备份污染、低覆盖率、无上下文感知等架构级问题，从零构建安全可靠的汉化引擎，并扩展翻译覆盖率至 99.3%，添加上下文感知架构和 CI 覆盖率回归检测。

## Milestone Summary

**Key Decisions:**
- 完全重写（非修补）-- 备份污染是架构级 bug
- 纯 Python 标准库，零外部依赖
- 增强型 regex（非 AST）-- 13MB minified 文件不适合 AST
- 三级替换策略：长(全局) / 中(引号边界) / 短(词边界+skip)
- Hook sed 替换集成到 Python 引擎（hooks.py）
- 自动翻译采用纯规则引擎（词典->精确匹配->UI模板->动词模式->前缀后缀）
- coverage 命令：表格+JSON 双输出，按字符串长度分类
- 翻译映射表 v6 格式支持上下文标签（同一英文多翻译）
- 上下文检测基于 10 种正则模式识别功能区块
- validate 命令检测三类质量问题（中英混杂/同义不一致/格式占位符丢失）
- CI 覆盖率回归检测：PR 不允许覆盖率低于主分支基线

**Issues Resolved:**
- 备份污染 -- v2.0 备份被中文污染导致无法恢复
- 路径检测 -- v2.0 仅支持 npm 全局安装
- 短字符串误替换 -- "Error" 等全局替换破坏运行时逻辑
- 版本更新后失效 -- v2.0 无版本检测，CLI 更新后汉化丢失
- 无量化能力 -- 无法知道翻译覆盖率
- 低覆盖率 -- v3.0 Phase 1-3 仅达 48%，Phase 5 扩充至 99.3%
- 无质量验证 -- 翻译质量全靠人工，Phase 5 添加 validate 自动检测
- 无上下文感知 -- 同一英文只能有一个翻译，Phase 4 支持按组件区分

**Technical Debt:**
- auto_translate 模块覆盖率仅 63%（规则复杂度高）
- prefix_suffix_match 产生低质量翻译（已弃用）
- CI 仅 GitHub Actions，未验证 macOS/Linux 差异
- validate 报告 342 个翻译质量问题（272 中英混杂 + 70 同义不一致），属后续优化项
- resolve_translation 函数在生产代码中未使用（replacer 内联等价逻辑）
- Phase 2-3 缺少 VERIFICATION.md（在验证系统建立前发布）
