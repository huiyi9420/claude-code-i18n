    **Step 1: 创建 coverage.py 命令模块**

    创建 `scripts/i18n/commands/coverage.py`：

    ```python
    def cmd_coverage() -> None:
    ```

    核心逻辑：
    1. 调用 `get_cli_dir()` 获取 CLI 安装目录（如果找不到会自动输出错误并退出）
    2. 加载翻译映射表：`get_data_dir() / MAP_FILE`，获取 `existing` 翻译集合
    3. 加载 skip words：`get_data_dir() / SKIP_FILE`，获取 `skipped` 集合
    4. 调用 `BackupManager(cli_dir).ensure_backup()` 确保备份存在
    5. 从备份读取内容，调用 `scan_candidates(content, existing, skipped, NOISE_RE)` 获取未翻译候选
    6. 计算覆盖率指标：
       - `translated` = len(existing)
       - `untranslated` = len(candidates)（scan_candidates 返回的候选数）
       - `skipped_count` = len(skipped)
       - `total` = translated + untranslated + skipped_count
       - `percentage` = translated / (translated + untranslated) * 100
    7. 按字符串长度分类（per D-02, COV-02）：
       - long: >20 字符的翻译条目数 / 候选数
       - medium: 10-20 字符
       - short: <10 字符
    8. 调用 `format_coverage_table(...)` 格式化表格输出（per D-02 表格形式）
    9. 同时调用 `output_json(...)` 输出结构化 JSON

    `format_coverage_table()` 函数：
    - 使用纯文本表格（不依赖 rich/tabulate 等外部库 — 零外部依赖约束）
    - 表格列：类别 | 已翻译 | 未翻译 | 覆盖率
    - 行：长字符串(>20) | 中字符串(10-20) | 短字符串(<10) | 总计
    - 输出到 stdout
